Optional Comments:

Handle NodeID collisions? – I wasn’t able to test this on a large scale DHT, so I don’t know if it does work properly.

 Display information on console when DHT node added – When a node is added it does display on its own console but not any other. I originally display in the root node along with the joining node but the display pops up in the middle of me doing something and it look messy so I took it out. You can check the status of the node by typing in “status’ and that will display its’ information

Runs test file, if provided – No test file was provided

Allow creation of up to 999 Mcast groups – You could theoretically implement 999 Mcast groups but I wasn’t able to test that so I’m not sure how the program will react. The most I tested was 3 Mcast group with 6 nodes.

The way I set up my Mcast groups is each node has a mGroup data type which contains the group id, a root status, and forwarder status. If the group id is non-zero the node belongs to that group. If the root status is true, that node is the root node for the Mcast group, and if the forwarder status is true, that node is just a forwarder its group id should be 0.
